# Демо-площадка упражнений

Эта страница — технический стенд для проверки компонентов в инлайн-режиме.
Здесь упражнения встроены прямо в текст, как они будут жить внутри статей.

Промпт-инжиниринг начинается с понимания того, что LLM не «ищет» ответ,
а **продолжает текст**. Проверим:

<MultipleChoice
  id="demo-llm"
  data={{
    question: 'Что на самом деле делает LLM, когда отвечает на запрос?',
    options: [
      { id: 'a', text: 'Ищет готовый ответ в базе данных' },
      { id: 'b', text: 'Предсказывает следующий токен за токеном' },
      { id: 'c', text: 'Копирует подходящий текст из интернета в реальном времени' },
    ],
    correct: ['b'],
    explanation:
      'Модель не хранит «ответы». На каждом шаге она оценивает вероятности следующего токена и выбирает один — так слово за словом рождается ответ.',
  }}
/>

Хороший промпт обычно собран из нескольких элементов: роль, контекст, задача,
формат и ограничения. Каждый из них влияет на ответ по-своему.

<MultipleChoice
  id="demo-role"
  data={{
    question: 'Какой элемент промпта задаёт модели её «личность» и экспертизу?',
    options: [
      { id: 'a', text: 'Роль' },
      { id: 'b', text: 'Формат вывода' },
      { id: 'c', text: 'Ограничения' },
    ],
    correct: ['a'],
    explanation:
      'Роль («ты — опытный редактор...») задаёт точку зрения и уровень экспертизы, с которых модель будет отвечать.',
  }}
/>

Если оба упражнения показывают «верно/неверно» и объяснение, а кнопка
«Попробовать снова» сбрасывает состояние — ядро Фазы 1 работает.
