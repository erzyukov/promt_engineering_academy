# 03. Контент-план

## Принципы интеграции теории и практики

- **Формат — MDX.** Статья пишется как обычный текст, но в любом месте можно вставить `<Упражнение/>`. Это и даёт «органичность»: упражнение стоит ровно там, где только что объяснили концепцию.
- **Правило «объяснил → проверил».** Каждый ключевой тезис закрепляется инлайн-упражнением **сразу после** абзаца, а не свалкой тестов в конце.
- **Разнообразие типов.** В рамках статьи чередуем форматы (выбор → соединение → порядок → поиск), чтобы проверка не была монотонной.
- **Финальный тест** в конце статьи (`<Quiz>`) собирает 4–6 вопросов по всему материалу с общим счётом и сводкой.

### Параметры статьи (ориентир)
| Параметр | Значение |
|---|---|
| Длина | 800–1500 слов (≈ 6–10 минут чтения) |
| Инлайн-тестов | 3–5 на статью, разных типов |
| Финальный тест | 4–6 вопросов, режим `quiz` |
| Разделов | 4–6 смысловых секций |

## Статья №1 — «Как работают LLM (на пальцах)» — полная структура

**Цель:** дать интуицию, как модель порождает текст, без математики. Снять магию → убрать страх «AI всё знает».

**Файл:** `src/content/articles/how-llms-work.mdx`

### Раздел 1. Вступление: AI — не Google
- Тезис: модель не «ищет ответ», а **продолжает текст**, предсказывая следующий кусочек.
- 🧩 **Инлайн `MultipleChoice`** (single): «Что на самом деле делает LLM, когда отвечает?» → варианты: ищет в базе / предсказывает следующий токен / копирует из интернета. Объяснение про предсказание.

### Раздел 2. Токены — кирпичики текста
- Тезис: текст дробится на **токены** (кусочки слов), модель оперирует ими, а не буквами/словами.
- 🧩 **Инлайн `FillTheBlank`**: «Токен — это не слово, а ___ слова». Принимает «часть/кусочек/фрагмент».
- 🧩 **Инлайн `TrueFalse`**: «Одно длинное слово может состоять из нескольких токенов» → Верно (+ объяснение про редкие слова).

### Раздел 3. Предсказание следующего токена
- Тезис: на каждом шаге модель оценивает **вероятности** следующего токена и выбирает один; так слово за словом рождается ответ.
- 🧩 **Инлайн `OrderSteps`**: расставь по порядку, что происходит при генерации: получает промпт → дробит на токены → оценивает вероятности → выбирает токен → повторяет. Проверяет понимание цикла.

### Раздел 4. Контекст и его пределы
- Тезис: модель «видит» только то, что в **контекстном окне**; что не влезло — для неё не существует.
- 🧩 **Инлайн `MultipleChoice`** (multi): «Что входит в контекст модели?» → системный промпт / твой вопрос / переписка выше / весь интернет в реальном времени. Частичный балл.

### Раздел 5. Почему отсюда растут и сила, и слабости
- Связка: предсказание + контекст → объясняет, почему AI бывает креативным, но и почему выдумывает (мостик к статье про галлюцинации).

### Финальный тест статьи
`<Quiz id="quiz-llm" title="Проверь себя: как работают LLM">` — 5 вопросов:
1. `MultipleChoice` — суть работы LLM.
2. `TrueFalse` — про токены.
3. `OrderSteps` — цикл генерации (повтор на новом материале).
4. `MatchPairs` — термин ↔ определение (токен, контекст, промпт, галлюцинация).
5. `MultipleChoice` (multi) — что входит в контекст.
→ Прогресс-бар, итог в процентах, `QuizSummary` с разбором по каждому вопросу и рекомендацией («перечитай раздел про …» при низком балле).

## Статья №2 — «Структура эффективного промпта» — тезисно

**Цель:** научить раскладывать промпт на части и осознанно их добавлять.

- **Разделы:** зачем структура → 5 элементов (Роль / Контекст / Задача / Формат / Ограничения) → разбор плохого vs хорошего промпта → итерация.
- **Инлайн-упражнения:**
  - `MatchPairs` — соедини элемент промпта с его назначением.
  - `PromptBuilder` — собери промпт из блоков по слотам Роль/Контекст/Задача/Формат.
  - `CaseStudy` — улучши слабый промпт; эвристический чеклист критериев.
  - ⭐ `PromptRefinementLadder` — пошагово добавляй элементы и смотри, как «ответ AI» становится лучше (кастомный компонент, кульминация статьи).
- **Финальный тест:** 5–6 вопросов (узнавание элементов, выбор лучшего промпта, true/false по ограничениям).

## Статья №3 — «Галлюцинации: почему AI уверенно врёт» — тезисно

**Цель:** научить не доверять вслепую и распознавать выдумки.

- **Разделы:** что такое галлюцинация → почему она неизбежна (следствие предсказания) → типичные признаки (ложная конкретика, несуществующие источники) → как проверять.
- **Инлайн-упражнения:**
  - `SpotTheHallucination` — найди выдуманный факт в «ответе AI» (продвинутый компонент, ядро статьи).
  - `TrueFalse` — мифы о галлюцинациях («если AI уверен — значит, прав» → Неверно).
  - `MultipleChoice` — какие признаки выдают галлюцинацию.
- **Финальный тест:** 4–5 вопросов, включая второй `SpotTheHallucination` на новом тексте.

## Реестр статей (`src/content/registry.ts`)
Единый список для главной страницы и навигации: для каждой статьи — `slug`, `title`, `summary`, `order`, `readingTime`, ссылка на MDX. Главная (`Home`) рендерит карточки из реестра; роут `/#/article/:slug` открывает нужный MDX.
